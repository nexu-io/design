# Open Design

This directory isolates the Open Design product design work from the rest of the repository.

## Contents

- `design-spec.md` captures the product UI direction and acceptance criteria.
- `implementation-notes.md` records how the current app implementation maps to this design package.
- `assets/` contains Open Design brand assets used by the product shell.
- `screenshots/` is reserved for visual QA captures.
- `patches/` contains source implementation patches when the target repository structure is not directly compatible.
